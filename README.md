# manual-gen

웹 시스템을 분석해 **사용자 매뉴얼(PPT/Word)을 자동으로 만들어 주는** Claude Code skill.

소스코드에서 화면 목록을 뽑고, 실행 중인 시스템이 있으면 브라우저로 메뉴를 돌며
실제 스크린샷을 찍은 뒤, 국내 SI 매뉴얼 관행 형식으로 문서를 완성한다:

```
[화면 스크린샷 — 기능 위치에 붉은 테두리 + ①②③ 번호 배지]
[사진 3] 사용자 목록

① 아이디, 이름으로 검색할 수 있습니다.
② [등록]을 클릭하면 등록 화면으로 이동합니다.
③ [삭제] 클릭 시 확인 후 삭제됩니다.
※ 관리자 계정은 최소 1개 이상 필요합니다.
```

표지 → 목차 → 장 간지 → 화면별 설명 슬라이드 구조의 `.pptx`(또는 `.docx`)가
프로젝트 폴더에 생성된다.

---

## 1. 준비물

| 필요한 것 | 비고 |
|---|---|
| **Claude Code** (데스크톱 앱 또는 CLI) | 이 skill의 실행 환경 — https://claude.com/claude-code |
| **Python 3.10+** | 문서 생성·이미지 처리 스크립트용 |
| Chrome (권장) | 화면 캡처용 — 없으면 Edge/Whale 등으로 선택 가능 |

파이썬 패키지(python-pptx, Pillow, Playwright 등)는 **처음 실행할 때 skill이 스스로
점검해서 없는 것만 설치를 안내**하므로 미리 설치할 필요 없다.

## 2. 설치

PowerShell에서 한 줄:

```powershell
git clone https://github.com/ycw3320/manual-gen.git "$env:USERPROFILE\.claude\skills\manual-gen"
```

**확인**: Claude Code에서 **새 대화**를 열고 입력창에 `/man`을 치면 `/manual-gen`이
자동완성 목록에 나타나면 설치 완료다. (설치 전에 열어 둔 대화에서는 안 보인다 — 새 대화 필요)

## 3. 빠른 시작

```
cd C:\projects\대상프로젝트     ← ① 매뉴얼을 만들 프로젝트 폴더에서 Claude Code 실행
/manual-gen                    ← ② 호출 (또는 "이 시스템 관리자 매뉴얼 만들어줘")
```

이후 자동으로 진행된다:

1. **환경·프로젝트 분석** — 의존성 점검, 기술 스택·화면 목록·실행 URL·대상 독자를 추론
2. **확인 1회** — 추론 결과 표를 보여주며 "이대로 진행?"을 묻는다 (유일한 필수 질문)
3. **캡처** — 브라우저로 화면을 돌며 스크린샷 + 테두리·번호 배지 합성 (로그인이 필요하면
   이때 "브라우저 창에서 로그인해 주세요"라고 요청할 수 있다 — 비밀번호를 채팅에 입력하지 않는다)
4. **문서 생성** — 프로젝트 폴더에 `관리자매뉴얼_<시스템명>_<날짜>.pptx` 생성
5. **검수 보고** — 캡처 못 한 화면(placeholder)·주의 사항을 정리해 알려준다

### 자주 쓰는 호출

```
/manual-gen                                    # 폴더 자동 인식, 확인 1회
/manual-gen http://localhost:8080 관리자 pptx    # 대상·독자·형식을 인자로 지정
/manual-gen http://localhost:8080 auto          # 질문 0회로 끝까지 자동
/manual-gen 이어서                               # 중단된 작업 재개
```

### 실행 중 나올 수 있는 질문

| 질문 | 언제 | 하면 되는 것 |
|---|---|---|
| "이 설정으로 진행할까요?" (표 1개) | 매 실행 1회 (auto 제외) | 확인 또는 수정 항목 선택 |
| "브라우저 창에서 로그인해 주세요" | 자동 로그인이 안 될 때 | 뜨는 창에서 직접 로그인 → 완료 알림 |
| "만들 영역: 하나만/전부?" | 관리자/사용자 화면이 나뉜 시스템 | 선택 (전부 = 영역별 문서 생성) |
| "화면에 개인정보가 보입니다" | 운영 데이터 감지 시 | 블러 처리/진행 여부 선택 |

무개입으로 쓰려면: `auto` 인자 + 환경변수 로그인을 설정한다 —
`$env:WUM_LOGIN_ID = "계정"; $env:WUM_LOGIN_PW = "비밀번호"` (현재 터미널에서만 유효,
기록에 남지 않음).

## 4. 대상 시스템 결정 기준

1. **명시 인자** — `/manual-gen C:\projects\myapp` 또는 `/manual-gen http://host:port`
2. **재실행 설정** — 현재 디렉토리의 `manual-work/config.md` (이전 실행 대상 재사용)
3. **현재 작업 디렉토리** — 웹 프로젝트 마커(pom.xml·build.gradle·package.json 등)가
   있으면 그 폴더가 대상 (실행 URL은 설정 파일 포트·로컬 서버 응답으로 추론)
4. 어느 것도 없으면 확인 단계에서 경로/URL을 질문

## 5. 산출물 위치

```
C:\projects\대상프로젝트\
├── 관리자매뉴얼_시스템명_20260714.pptx   ← ★ 최종 문서 (프로젝트 루트)
└── manual-work\                          ← 작업 폴더 (지우지 말 것 — 재개·재생성의 기준)
    ├── config.md        설정            ├── screenshots\  캡처 원본·배지 합성본
    ├── inventory.md     진행 원장        ├── sections\     화면별 원고 조각
    └── manual-draft.md  병합 원고
```

- 독자 영역이 여러 개면 `시스템관리자매뉴얼_...`, `기관관리자매뉴얼_...` 처럼 영역마다 별도 파일
- 대상 프로젝트가 git 저장소면 `manual-work/`를 `.gitignore`에 추가할 것(스크린샷에 데이터 포함 가능)

## 6. 참고 템플릿(조직 서식) 사용

이미 쓰는 PPT 서식 틀이 있으면 **서식은 그대로 두고 내용만 채우는** 방식이 가능하다:

```
/manual-gen http://localhost:8080 관리자 pptx 템플릿은 C:\양식\회사매뉴얼서식.pptx
```

- 템플릿의 표지/목차/간지/화면 슬라이드 서식을 식별해 화면 수만큼 복제하고,
  텍스트(절 번호·개요·번호별 설명·캡션)와 스크린샷만 교체한다. 템플릿에 번호 배지
  도형이 있으면 그 도형을 재사용한다.
- **제약**: 서식 복제는 Claude Code **데스크톱 앱**에서만 동작한다(전용 pptx skill 필요).
  CLI에서는 템플릿 없이 내장 레이아웃으로 생성된다.

## 7. 자주 겪는 문제

| 증상 | 원인·해결 |
|---|---|
| `/manual-gen`이 목록에 안 뜸 | 설치 후 **새 대화**를 열어야 인식된다 |
| 스크린샷에 테두리·배지가 없음 | 캡처를 건너뛰고 문서만 재생성한 경우 — "배지 적용해서 재생성" 지시 |
| 로그인 화면에서 멈춤 | 캡처용 브라우저 창에서 직접 로그인 후 "로그인했어"라고 알리면 이어서 진행 |
| 중단된 작업을 이어서 하고 싶음 | 같은 폴더에서 `/manual-gen 이어서` — inventory 기준으로 재개 |
| 원고는 그대로 두고 문서만 다시 뽑고 싶음 | "pptx만 재생성해줘" — 캡처 재사용, 수 초 소요 |
| 특정 화면이 placeholder(회색 상자)로 나옴 | 캡처 실패 화면 — 검수 보고의 사유 확인 후 해당 화면만 재캡처 지시 |

## 8. 구성 (내부 구조)

```
SKILL.md                        # 워크플로 본체 (모드 판정 → 인터뷰 → 인벤토리 → 캡처 → 문서화 → 산출)
references/
├── source-analysis.md          # 스택 감지 표 + 스택별 화면/메뉴 추출 힌트 + 독자 영역 판별
├── live-capture.md             # 캡처 수단 의사결정 트리 + 배지·블러 절차 + 안전 수칙
├── manual-template.md          # 표준 목차·화면 문서화 포맷·표기 규약
└── output-formats.md           # docx/pptx 규격 (고정 프레임·그림자·분할·검수)
scripts/
├── cdp_capture.py              # 캡처: 풀페이지·배지 좌표(--mark)·PII 블러(--redact-*)·
│                               #   에러 화면 감지·브라우저 검증·자동 로그인·재개(--skip-existing)
├── capture_browser.ps1         # OS 수준 창 캡처 (Windows, 의존성 없음)
├── annotate_screenshot.py      # 번호 배지+강조 테두리 합성
├── resize_images.py            # 문서 삽입 전 이미지 축소
├── draft_parser.py             # 원고 공용 파서
├── validate_draft.py           # 원고 사전 린터 — 구조 위반을 빌드 전에 차단
├── check_env.py                # 환경 점검(doctor) — 의존성·브라우저·가능 경로 확인
├── build_pptx.py               # 원고→PPTX (표지/목차/고정 프레임·그림자·분할·자체 검증)
└── build_docx.py               # 원고→DOCX (표지/자동 목차/헤딩/캡션/페이지 번호)
```

주요 동작 원칙: 하이브리드 분석(소스+라이브, 한쪽만으로도 동작) · 독자 영역 자동
분리(영역별 별도 문서) · Chrome 최우선 캡처 · 무개입 로그인 3단계 · 어떤 환경에서든
동일 구조 산출(번들 빌더 기본 + 원고 린터 + 기계 검수).

## 9. 안전 원칙

- 라이브 순회는 **읽기 전용**: 저장/등록/수정/삭제/제출 버튼을 클릭하지 않음
- 비밀번호를 채팅으로 받지 않음: 세션 재사용/환경변수 기반 자동 로그인, 그 외에는
  사용자가 브라우저에서 직접 로그인 (자격 증명은 대화·저장소에 미노출)
- 화면에 개인정보(실명·연락처 등)가 보이면 캡처 전 확인, 캡처 시점 블러
  (`--redact-file`/`--redact-email` — PII를 CLI 인자로 받지 않으며, 가릴 문자열
  목록 파일은 `.gitignore` 영역인 `manual-work/`에 둔다)
