# 출력 형식별 변환 명세 (docx / pptx)

manual-draft.md와 screenshots/를 최종 문서로 변환할 때의 요구 명세.
생성 자체는 각 형식의 전용 skill(`anthropic-skills:docx`, `anthropic-skills:pptx`)을
Skill 도구로 호출해 수행하고, 이 명세를 그 호출의 요구사항으로 전달한다.

## 공통

- 산출 파일명: `사용자매뉴얼_<시스템명>_<YYYYMMDD>.docx|pptx`
  (관리자용이면 `관리자매뉴얼_...`). 저장 위치는 대상 프로젝트 루트(또는 사용자 지정 경로).
- 이미지는 `_annotated.png`가 있으면 그것을, 없으면 원본을 쓴다.
- 사전 처리로 `python scripts/resize_images.py manual-work/screenshots --max-width 1400`을
  실행해 둔다 — 원본 해상도 그대로 넣으면 파일이 수십 MB로 커지기 때문이다.
- placeholder 블록은 회색 배경의 안내 상자("화면 이미지 추후 삽입 — SCR-ID 화면명")로
  렌더한다. 조용히 누락시키지 않는다.

## docx 변환 명세

| 요소 | 명세 |
|---|---|
| 표지 | 시스템명(제목), "사용자 매뉴얼"/"관리자 매뉴얼", 버전, 작성일. 페이지 번호 없음 |
| 목차 | 자동 목차(TOC 필드) — 헤딩 1~3 수준까지 |
| 헤딩 계층 | Heading 1 = 장(01. 시스템 개요) / Heading 2 = 절(3.2 회원 관리) / Heading 3 = 하위 절(3.2.1 관리자) |
| 본문 글꼴 | 맑은 고딕 10~11pt (한국어 문서 표준) |
| 스크린샷 | 페이지 폭 맞춤(여백 제외 100%), 가운데 정렬. 이미지 아래 캡션 문단 "[사진 N] 화면명" 9pt 가운데 |
| 번호별 설명 | ①②③ 원문자 그대로 문단 목록으로 (자동 번호 목록 스타일로 바꾸지 않는다 — 배지와 문자가 일치해야 하므로) |
| ※ 주의 | 별도 문단, 굵게 또는 음영 처리 |
| 표 | 머리행 음영, 테두리 실선 (회원체계 표 등) |
| 머리글/바닥글 | 머리글: 시스템명 · 문서명 / 바닥글: 가운데 페이지 번호 |
| 페이지 | A4 세로, 여백 상하 2cm 좌우 2.5cm. 각 장(Heading 1)은 새 페이지에서 시작 |

## pptx 생성 명세

### 공통 규격

- 16:9 (13.333 x 7.5 in). 한국어 글꼴 맑은 고딕.
- 슬라이드 구성: ① 표지 → ② CONTENTS → ③ 장 간지(01/02/03...) → ④ 화면 슬라이드(본체) → 부록.
- 화면 슬라이드는 **슬라이드 1장 = 화면 1개(inventory 기능 단위 1개)**가 원칙.
  번호별 설명이 6개를 넘으면 같은 제목으로 슬라이드를 나눈다(제목에 (1)/(2) 표기).

### 화면 슬라이드 레이아웃 (신규 생성 시)

```
┌─────────────────────────────────────────────────┐
│ 03. 화면 및 기능 설명        3.2.1 관리자   (p)  │ ← 상단 헤더: 장 제목(좌) + 절 번호·라벨(우), 페이지 번호
├─────────────────────────────────────────────────┤
│ 화면 개요 1~2문장 (14pt)                          │
│                                                   │
│ ┌───────────────────────────┐  ① 설명 …          │
│ │                           │  ② 설명 …          │
│ │   스크린샷(annotated)      │  ③ 설명 …          │ ← 좌: 이미지(높이 우선 맞춤)
│ │                           │  ※ 주의 …          │    우: 번호별 설명(12pt)
│ └───────────────────────────┘                     │
│  [사진 N] 화면명 (9pt, 이미지 하단)                │
└─────────────────────────────────────────────────┘
```

- 이미지가 가로로 긴 화면(목록 위주)은 상(이미지)/하(설명) 배치로 전환한다.
- 표지: 시스템명 크게, "관리자용/사용자용" 뱃지, 연도. 장 간지: 장 번호+제목만 크게.
- CONTENTS: 장·절 목록 + 슬라이드 번호. 생성 마지막에 실제 슬라이드 번호로 갱신한다.

### 참고 템플릿 pptx가 제공된 경우 (서식 복제)

조직의 기존 매뉴얼과 형식을 일치시키는 것이 목적이므로, 새 레이아웃을 만들지 말고
템플릿의 서식을 그대로 복제한다:

1. pptx skill의 편집 워크플로로 템플릿을 unpack한다.
2. 슬라이드 유형을 식별한다: 표지 / CONTENTS / 장 간지 / **완성도 높은 화면 슬라이드 1장**(서식 원본).
3. 화면 슬라이드 서식 원본을 화면 수만큼 복제하고, 텍스트(절 번호·라벨, 개요, 번호별
   설명, 캡션)와 이미지(스크린샷)만 교체한다. 번호 배지가 템플릿에 도형으로 있으면
   그 도형을 재배치해 쓰고, annotate_screenshot.py 합성은 생략한다(이중 배지 방지).
4. CONTENTS의 절 목록·페이지 번호를 실제 구성에 맞게 갱신한다.
5. 템플릿에 있던 예시 내용이 남지 않았는지 전수 확인한다(placeholder 텍스트 grep).

### 검수 (두 형식 공통)

- 변환 후 각 형식의 skill이 제공하는 QA 절차(텍스트 추출 대조, 시각 검사)를 수행한다.
- 특히 확인: 이미지 누락/깨짐, 캡션-사진 번호 대응, 목차-본문 번호 일치, 한글 깨짐.
